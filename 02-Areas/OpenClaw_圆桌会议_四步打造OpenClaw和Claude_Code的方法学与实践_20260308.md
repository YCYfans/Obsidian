# 四步打造OpenClaw和Claude Code的方法学与实践

> 标签：OpenClaw / Claude Code / AI Agent / 数字员工 / 个人效率

---

## Summary

本文分享了作者从24年底到26年初，从Cursor Agent到Claude Code再到OpenClaw的实战经验。文章围绕"什么人在AI时代的杠杆会被放大，什么人会被淘汰"这一问题，通过四步方法论阐述了如何打造和管理AI数字员工：

1. **准备工作**：推荐Mac Mini作为AIPC，搭配模型API（Kimi K2.5/Claude）
2. **入门开胃**：三个OpenClaw实战案例——个人秘书、语音盯盘工具、咨询助手
3. **Agents世界观**：区分本地Agent（正式员工）vs 云端Agent（外包员工），强调向上管理和向下管理的重要性
4. **向下管理实践**：手机端远程管理Claude Code、语音交互提效、让AI给领导做选择题

**核心观点**：AI时代会当领导的人杠杆会被放大——既要向上管理获得真实业务需求，也要向下管理有效组织Agents团队。

---

## 前言

从24年底开始入手Cursor Agent，到近两个月沉浸Claude Code的并行开发，到最近的OpenClaw养了六只虾，当前AI的奇点带来的冲击和震撼远超我的预期。年前大领导问了个问题：什么人在AI时代的杠杆会被放大，什么人会被淘汰。在26年开坑和大家进行思想碰撞，在实践中试图回答这个问题。

第一篇分四步分享下当前Agents的实践。（以下内容用每天下班1-2小时可完成，适合中高强度打工牛马）

---

## Step 1 准备工作：缓解焦虑最好的方法是花钱买装备

我从身边很多朋友，包括领导身上都感受到了焦虑，尤其是看到社媒上大家动辄开数十个Agents数字员工7*24打工。怎么解？花钱买买买咯。

其实人和公司是一样的，互联网大厂缓解焦虑FOMO的方法是提升资本支出（Capex），买够了GPU就一定不会被时代淘汰。对于个人，那就是开箱一台专属运行OpenClaw的AIPC，毕竟男生买电子设备和女生买包一样上瘾。

当前AIPC的核心是生态完整的PC，而不是本地AI。最豪华的DGX Spark不过跑个200B的fp4模型，效果、速度上都完全不够看，代价却是ARM版本Ubuntu的生态，没有Chrome、Office和notion，也没有网关软件版本。在短期没有本地模型落地的预期下，标配Mac Mini搭配模型API（推荐Kimi K2.5和Claude）。

所以2个月不到我果断把崭新的DGX Spark放在旁边又入手了个Mac Mini，血汗钱给大家买个教训（感谢赞助Spark的金主爸爸）。

当然这是现状，AI是天上一天，地下一年，我认为一年的时间业界会把主流千亿大模型做到本地化执行，通过激进的diffusion/投机把TPS打到20甚至50。

**小结：当前最好的AIPC是没有AI算力的Mac Mini**

---

## Step 2 入门开胃：OpenClaw（龙虾）能做些什么

装备到位了，首先按照官方指南把OpenClaw接了飞书，所有任务就可以通过语音下发。要想让OpenClaw和其他Agent有所不同，那干的事情必须得是高度依赖你本地权限干的事儿，下面给大家打个样~

### 龙虾1号：给自己做了个秘书

最近领导要更新下内部简历，加上我要申请旅游的签证。而春节在外度假的我，这两个事儿我就都远程交给OpenClaw完成了。因为OpenClaw我本地的文件权限，有以前的简历和材料，还能用headless浏览器去找我的Google Scholar，以及签证网站的材料要求，完成度基本有80%。

当然，由于内部简历和求职简历要求不一样，以及最新的工作经历还没做好数字化，所以这部分还是把要补充的信息语音告诉OpenClaw让他完成的，之后还花了半个小时手动改。这就是慢慢养虾的过程了。

### 龙虾2号：给我老妈做了个语音盯盘

老妈是个老韭菜，帮她搞了一个通过飞书语音盯盘下单工具，比如："每10分钟查下NVIDIA的股价，涨幅>=5%下单100股"。这完全算不上Agent，但她老人家还蛮开心的，毕竟可以专心刷剧，自动下单还有飞书提醒。

然而，它解决的只是精力问题，是个通往赌场的时空门，毕竟股票不会因为你下单更智能就涨。有精力的同学，就要在这个基础上调教龙虾去钻研行业研究和技术面操作了。

### 龙虾3号：给领导做个咨询助手

领导经常会给你发些咨询，比如《DeepSeek发布梁文锋署名新论文开源相关记忆模块Engram》，《智谱GLM-5技术全公开！完全适配华为等国产芯片》。这些信息有序的散布在微信，我司工作软件，甚至是小红书的不同账号上。

而龙虾3号偷偷溜进了一个只有我和我领导的飞书群（下班用），同时有一个网页客户端（上班用，因为不能访问飞书这些IM）。这样所有信息汇总在一个地方，再也不用想是在哪个平台看到这个信息了，可以每天发个完整的咨询日报。

（OpenClaw接到飞书群里必须要企业账号，个人用户折腾了好半天没搞定，差评）

**小结：数字化（信息/数据/接口）是繁琐沉淀资产的过程但也是智能化前的必经之路**

## Step 3 进阶前的思想准备：Agents的世界观

这里你可能会问：为什么一定是Mac Mini，各家云和模型的龙虾行不？额，还真不行。

世界上只有两种Agent，跑在本地的OpenClaw（正式员工）和云端的Agent-as-a-Service（外包员工），包括各家的Claw（KimiClaw/MaxClaw）和各家的Agent（Manus/Kimi/豆包）等等。

两者都可以跑7*24的任务，但前面两只龙虾我只敢放在本地养。你可能会说，最后不都是模型API调用吗？是的，但我只敢把护照放在我的Mac Mini，不敢上传，也就是只有本地的OpenClaw才可以帮我去填申请签证。当然，如果你的Agent是做不需要任何的隐私信息的工作，无脑选云端Agent。

然后你可能说，就这？这都能算Agent吗？能帮我赚钱吗？确实，上面这三只龙虾是开胃。

回到开题年前大领导问的问题：什么人在AI时代的杠杆会被放大，什么人会被淘汰。我给的答案是——**会当领导的人**。本质上是会向上管理和向下管理。

### 向上管理

向上管理指的是面向客户和领导，获得真实的、有价值的业务需求，让自己的token烧得有意义。如果你当前还没找到这样一个业务，那这里抛个砖，对于一个上班10+小时的大厂牛马，时间是最宝贵的资源。因此，最有价值的任务是让数字分身分担部分我们的工作，用最少的时间完成更多高价值的任务。

关于数字分身的完整故事，我们后面单独聊~

### 向下管理

向下管理，能够有效地组织你的Agents员工，明确任务，高效完成并验收。怎么用领导带团队的思维，让AI员工高效运转。

你可能发现了，AI一定程度上是个马太效应的放大器，掌握真实需求和资源的人，以前需要雇很多高成本员工来落地，现如今只需要一群Agents，同时满足忠诚、能干、不贵的不可能三要素，而人类牛马很容易失去创造独特价值的能力。能赚钱的人更能赚钱，赚不到钱OpenClaw也很难替你赚钱。

## Step 4 除非你能把Agents玩的很溜：向下管理的实践

凡事总有个例外。即便你尚未掌握向上管理的资源，但精通向下的Agents管理同样能让你紧跟时代的步伐。

着手向下管理的前提是，你已经拥有一个值得投入资源的业务。如果你还没有，那么你可以效仿我的做法：将自己塑造成一个工作分身，其价值上限等同于你的薪资水平。

### 向下管理提效实践之一：让Claude Code牛马手机端随叫随到

为了能够在机场、路上，甚至开会间隙都能让Claude Code迅速拿到我想让他干的事情，我做了一个网页工具，能够远程管理我的Claude Code项目。然后用在手机端（iPhone和Mate）把网页包装成了一个App，踢掉了我的相机位，方便随时打开。

打开后可以看到我当前Claude Code的项目，选择一个项目，直接输任务，对应的Claude Code就会去执行，然后可以打开任务查看执行结果。这里把Claude Code做成非交互式的组件，所以只要起一个Python用subprocess用下面的命令去调度Claude Code：

```bash
./claude -c -p [prompt] --dangerously-skip-permissions
```

有几个细节：
- ./claude是因为我每个项目的claude脚本都有一些定制化
- -c是为了继承之前的context
- --dangerously-skip-permissions是越过所有权限确认，我目前是个纯粹的激进派，效率远大于安全

当然，这个手机工具也可以直接支持新建项目、选模型，以及多人合作。

### 向下管理提效实践之二：领导有事都是打电话

人的主要瓶颈是信息传输的IO速率。如果我们打字的速率是1字/秒，假设1字=1 token，对应7168维的向量，那IO速率就是7KB/s，感人不。

唯一有效的提升方式就是动嘴，能提升3倍左右的速率，这是为什么领导有事儿从来都是一个电话。Claude Code创始人Boris也建议尽量用语音输入代替打字。这是在Human2Agent（H2A）的过程。

语音代替文字的道理同样适用于Agent2Human（A2H）。领导接受输入的方式，是听汇报，而不是看文档。

一个具体的案例是，我的数字分身的日报功能，一开始和很多小伙伴一样都拿Agent整理成文字版，早8点定时发送，但经常一忙就不看了。于是我就把每天的日报接上了豆包语音的API，每天上班开车的时候直接给我电话汇报。

汇报和AI播客的本质区别在于可以打断交互的，就像老板在汇报时会随时打断你提问，或者给你一顿洗脑，这就是老板养我、我养虾的过程。

### 向下管理提效实践之三：选择题要交给领导做，还要给领导讲清楚

每次看到Claude Code做出一个不尽如人意的产品设计时，脑袋里总能想起某领导的"教诲"：你作为SE（技术负责人）职责是给选项的，凭什么就替领导把决策做了？

所以，为了让我的数字分身本分一点，我给他加入了一个config和一个skill。

首先是/config里的Preferred output style，选择Explanatory，让Agent把他做的工作讲清楚，而不是只给结果。

然后是个skill我暂时叫/product-decision，目前还在打磨，核心是在要Claude Code在重要的产品特性上要给出3-4个选项，用html展示讲清楚，交给我来决策。

## 写在后面：个性化时代的开始，每个人从现在开始养自己的"虾"

很多人都会有同样的感受：永远也找不到一个用的顺手的笔记应用。

a16z说Vibe Coding带来的是软件的Youtube时刻，胡渊鸣老兄也在文章中提到"标准化软件的终结"。在生产力爆发的趋势下，大家不必再用同一个高昂开发成本下的某一款软件，而是都可以有自己个性化的软件。

这个事情其实不仅限于软件，比如，最近我把一首喜欢听的歌拿Suno改变成微醺定制版，一个只有我一个人听的歌，也能单曲循环一个月。

再比如今天的主题Agent。Claude Code的产品经理Cat在访谈里说，Claude Code is built to be customizable。每个人都会或早或晚开始自己的养虾之旅，打造自己的Agents，就像互联网时代人人开始做自媒体一样。

而注定世界上也不会有两只相同的虾，所以不必焦虑，一起加入养虾大军吧:)
