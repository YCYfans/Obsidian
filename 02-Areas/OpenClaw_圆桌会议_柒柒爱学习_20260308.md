# 💎和龙虾的基本共生心态

  

1. ## Cursor+Gemini /Claude Code能实现的已经很多了，为什么还要让龙虾做？
    

  

我开始用龙虾的第一天就在思考这个问题，和 GPT Gemini 探讨之后，综合以下答案：

当任务具备 **长期运行 + 事件触发 + 多步骤编排 + 自主纠错** 这些特征时， OpenClaw 通常比 Gemini + Cursor 更省工程成本。

  

|   |   |   |   |
|---|---|---|---|
|任务类型|核心特征|典型例子|OpenClaw 优势本质|
|长期后台监听类|持续运行；事件触发；自动执行|音频自动转写中心；下载文件自动分类；目录新文件自动摘要|内置 agent loop；原生支持持续执行；不依赖 IDE 存活|
|自动错误恢复类|易出现格式/API 异常；需根据报错重新决策|数据清洗与异常修复；自动调用 FFmpeg 转码后重试|内置“观察→推理→再执行”循环；无需自行设计 retry 框架|
|多步骤链式编排类|多环节串联；需要状态保持；跨多个工具|音频→转写→总结→分类→存笔记→发提醒；数据抓取→清洗→分析→生成报告→推送|原生 multi-step agent 结构；内置状态管理|
|自主决策型任务|处理路径不确定；需根据中间结果改变策略|根据文件类型决定处理方式；根据内容判断是否进入下一流程|内置决策循环；支持动态路径选择|
|多工具协调执行类|同时调用 shell、脚本、API、文件系统|同时调用 Python、Shell、外部 API 进行联合处理|默认 tool-calling runtime；执行环境已封装|

---

Cursor 更擅长：

> “在指定的范畴里写”

OpenClaw 更擅长：

> “持续执行 + 协调 + 自主处理”

  

  

他们之间的区别 Be like:

  

**1️⃣ 作家 vs 管家**

- **Cursor** 像一个顶级作家 你给它材料，它能把文章、代码、逻辑写得非常漂亮。
    
- OpenClaw 像一个全天候管家 你只说目标，它会自己去收集材料、跑腿、整理、归档。
    

写东西 → 找 Cursor

帮你把事情跑完 → 找 OpenClaw

**2️⃣ 建筑设计师 vs 全屋施工队**

- Cursor 是建筑设计师和单房间施工者 它帮你画图纸（写脚本、设计逻辑）。
    
- OpenClaw 是全屋施工队 它天天在不同的房间里干活，按图纸持续执行。
    

Cursor 负责“指挥和设计”。

OpenClaw 负责“真的去做”。

2. ### 第一天就构建复利工程 给龙虾制造进化加速度
    

  

为什么有的人养的虾第一天就很强大，有的人养了一个月之后还是小小虾，这里使用者背后的框架思维起到很大作用。

我们想让龙虾构建的loop是这样的（感谢我老公的分享）：

6. ### 养成良好每日复盘提升机制
    

  

我的龙虾每天做各种任务，会出各种各样的错误。为了让它自己吸取教训，不断变强，需要一段提示词，让它每天从错误和失败中吸取教训，不断提升

  

```Plain
【系统指令：每日自我进化协议】

你每天必须执行以下学习循环：

1.  错误记录 
   - 每次任务失败、报错、或结果不达预期时，立即记录：
     - 错误类型（网络/权限/逻辑/工具/理解）
     - 根本原因（你分析的真实原因，不是表面现象）
     - 当时的情境（用户指令、环境状态、可用工具）

2.  模式识别 
   - 每天结束时，回顾当天的所有错误
   - 找出重复出现的错误模式
```

当然你也可以让龙虾自己去根据关键词自己去搜，一步到位。但是，again，GPT 对我的了解还是远大于龙虾对我的了解。GPT 前几轮对话也是免费的，龙虾从第一步开始就要烧 token fee。可以一点一点逐步构建复利工程。当然也有可能我把龙虾搭建的很强壮的时候，新的鱿鱼螃蟹又要来了。

### **3.基础心态3：什么要自己参与 什么要给 AI 空间**

  

知道哪些东西不让 Agent 做，也是一种智慧。人和人交往，真诚最重要。诚意就代表着要付出一定程度上的人类时间和心力，背后也牵扯到人的信用。

拿公众号文案自动化发布来说，我把一篇文章认认真真审完了再发，是我的态度，我不希望有错别字，也不希望有严重的 AI 幻觉来误导读者。就算是 Opus 4.6 也还是有幻觉。如果我全自动发布的话，效率高是很高，但是如果误导别人，我就要承担背后的结果。所以，要用自己的信用担责的，就要自己去参与，尤其是核心的判断和决策。

AI 和人类共生的时代，我们很重要的一个能力，就是知道每个行业/每个工种，哪些地方的关键环节需要要被识别出来交给我们人类去做判断，让我们人类去对标准和结果负责，然后，我们可以不断收束，让 AI 的判断力变得更强，AI 不断能做更多的判断和决策，一点一点，共同成长。

AI agent，不管是龙虾还是其他， 可以是一个很好的合作者，但是目前不能 100% 托付给它。

  

  

### **4.基础心态4：龙虾不是全部 Token 也不是免费的**

  

GPT 5.4 已经具备操作本地电脑的能力。现在确实很贵，但是我们可以再观察。思考解法的出发点是，我怎么样让 Gemini Chat + Codex + Claudo Code + Cursor + 龙虾一起来协作做好， 而不是什么事情全部都让龙虾来做。不要轻易龙虾中毒。

如果自己对 agents 对 skill 还没有那么熟，或者还有很多需要科普的问题，可以先用免费的 Gemini 和 GPT 完成初步答疑，构想和探索，再让龙虾去做，这样会比较省钱。

  

  

### 5.基础心态5：学会一步到位 试探 AI 极致能力

  

我以前跟龙虾沟通是这样的： “可不可以麻烦你帮我看一下这个，做个这个”，

“你想想怎么办呢？”

随后龙虾可能会报错，告诉我应该去终端里做一二三。然后我又傻乎乎地去终端里做一二三，遇到问题又回来截图给他问怎么办。

  

有一天我老公打断我施法，他说你这样不行，你这样来回折腾，浪费的都是你的 token 费。直接让他干完，一步到位。然后他告诉了我他的独门沟通秘诀，就是当霸总：

“我只要结果我不要过程”

“你直接一步到位 全部完成”，

“你来干，别让我干”

  

当时我的世界观就被打开了。然后我开始学着当霸总，逼龙虾一步实现结果，而不是拆成三步。发现效果还挺好的。

  

这里有一个重要的思路，我觉得是值得吸收的，就是尽可能让 AI 解决 AI，让 AI 带动 AI，让 AI 修理 AI，别总是想着自己去修理（前提是我有便宜的 api）。

龙虾自己把自己折腾挂了的情况，我一律都让 Claude Code 去修理它。我之前换 API 供应商的时候，龙虾突然挂了，不回我消息，遇到这种情况，我通常让 CC 去修复它，CC 很快就可以搞定，我就只做甩手掌柜。

  

另外，模型本身在进化。之前分步骤让 Gemini 来做，是为了保证每一步的结果都更好，更准确。但是现在模型本身在飞速进步，所以有的时候确实可以让它一步到位，现在 CC 经常可以做到一步到位，效率确实提高了，只是偶尔会崩。

# 💎常用的 7 个龙虾 /命令（更多指令参见本 wiki 的子文档）

  

如果你在龙虾对话界面里面输入/，就会看到这些不同的指令（Slash Commands）。这些 `/` 指令（Slash Commands），本质上是 **AI Agent 的“捷径入口”与“系统级元数据透出”**。

在龙虾（或类似的 AI 平台）中，这种设计并非为了耍帅，而是为了解决大模型在 IM（即时通讯）环境下的一些天然痛点。

  

![](https://hcnrgdmax0fa.feishu.cn/space/api/box/stream/download/asynccode/?code=ZmZkZmQzNzVlNGZhMjA1ZmNkMjFhMjY4ZDNhY2EwNDBfcGVBMmVUS09Zb2V0OVdDRkEySkZOaHRObndwUWZEc0xfVG9rZW46Q0R6MGJwWWNnb0Y0a294V2s0NmNDQkZ5blNiXzE3NzI5ODEzMzk6MTc3Mjk4NDkzOV9WNA)

  

💡OpenClaw 命令 Top 详解：

  

• 日常高频：/status、/stop、/reset

• 任务管理：/subagents、/kill、/steer

• 安全控制：/approve

• 快速查询：/skill、/context

  

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  

1. /status — 会话仪表盘
    
      
    

• 功能：一键查看当前会话的用量、成本、模型、运行时长

• 价值：随时掌握 token 消耗，适合成本敏感用户

  

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  

2. /skill — 调用预设技能
    
      
    

• 功能：运行已配置的技能包（天气、GitHub、1Password 等）

• 价值：复杂操作一键调用，不用重复描述流程

• 用法：/skill weather 上海

  

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  

3. /stop — 紧急叫停
    
      
    

• 功能：立即终止当前运行的任务

• 价值：我跑偏了、卡住了？一键停止，不浪费时间和 token

  

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  

4. /subagents — 后台任务监控
    
      
    

• 功能：查看当前会话派出的子代理状态

• 价值：同时跑多个后台任务时，看谁在干活、进度如何

  

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  

5. /reset — 清空上下文
    
      
    

• 功能：重置当前会话，清空对话历史

• 价值：话题切换太大、上下文太乱时一键重置

• 注意：MEMORY.md 不会丢失，只是当前对话清空

  

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  

6. /approve — 安全审批
    
      
    

• 功能：批准或拒绝敏感操作请求

• 价值：删除文件、发群消息等操作需你手动确认

• 用法：/approve 查看、/approve y 批准、/approve n 拒绝

  

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  

7. /steer — 实时引导
    
      
    

• 功能：向正在运行的子代理发送指导指令

• 价值：后台任务跑一半想调整方向？不用停掉重来
